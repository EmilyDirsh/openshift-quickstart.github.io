What's a Quickstart?

A quickstart is an application configured to run on OpenShift, along with some metadata that tells OpenShift which cartridges it needs.

Quickstarts automate a number of steps you would otherwise have to do manually:

  1. Create a new application with a specific runtime
  2. Add a database cartridge and any other required cartridges
  3. Clone the application repo locally
  4. Overwrite the default repo with a pre-configured application
  5. Push the new code to OpenShift

A quickstart is not a cartridge:

  • a quickstart can depend on multiple cartridges, so it's useful for things like applications that require both a runtime and a database
  • quickstarts don't have as much functionality or flexibility at the system level as cartridges - they have to rely on their required cartridges for that.
  • quickstarts are less complicated, so easier to build than cartridges
  • quickstarts do not receive any automated maintenance or security updates

Use a Quickstart

With RHC

If you have rhc installed and setup, you can create a new application from a quickstart with the rhc app create command.

First, you'll need to know

  1. The URL for the quickstart's git repo
  2. The cartridges the quickstart requires

Then, you simply run the following command, substituting the above information: 

rhc app create [app name] [cartridges] --from-code=[git url]

As an example, here's how you can launch a new application using the meteor quickstart: 

rhc app create meteor nodejs-0.10 mongodb-2.2 --from-code=https://github.com/openshift-quickstart/openshift-meteorjs-quickstart.git

With the console

The OpenShift Online web console has a number of quickstarts available as options when you create a new application.
